2.3 Строгий режим - "use strict"

Строка "use strict"; добавляет поддержку старого кода

Должна находиться в начале скрипта в котором может быть старый код. Обычно его используют на весь файл, но можно поставить внутри конкретной функции.

Например задание восьмеричных переменных.

2.4 Переменные

let

Переменная объявляется через let, мы можем либо задать её сразу (let a = "a";), либо позже (let a; a = "a";).

Можем объявить несколько переменных в одной строке: (let a = "a", b = "b", age = 22;), но такой способ не рекомендуется.

let можно переопределять: (let a = "a"; a = "b";), старые данные удаляются.

Переменную let с именем можно задать всего один раз (let a = "A"; let a = "B"; => Ошибка).

var

Есть устаревший вид переменной let (var), которая имеет некоторые отлчия.

Переменную var с именем можно задать несколько раз (затерев старое значение) (var a = "A"; let a = "B"; => a === "B").

Пример блочной области видимости let

if (true) {
let hi = "Hi!"; console.log("2.4 let in", hi);
}
console.log("2.4 let out", hi);
// Ответ (см. лог): 2.4 let in Hi! и второй лог Uncaught ReferenceError: hi is not defined

2) Константы в camelCase используют во всех остальных случаях.

Придумывайте правильные имена

1) Название переменной должно иметь ясный и понятный смысл, говорить о том, какие данные в ней хранятся.

2) Используйте легко читаемые имена, такие как userName или shoppingCart.

3) Избегайте использования аббревиатур или коротких имён, таких как a, b, c.

4) Делайте имена максимально описательными и лаконичными. Примеры плохих имён: data и value. Такие имена ничего не говорят. Их можно использовать только в том случае, если из контекста кода очевидно, какие данные хранит переменная.

5) Договоритесь с вашей командой об используемых терминах. Если посетитель сайта называется «user», тогда мы должны называть связанные с ним переменные currentUser или newUser, а не, к примеру, currentVisitor или newManInTown.

2.5 Типы данных

Основы

1) Есть восемь основных типов данных в JavaScript: number, BigInt, string, bool, null, undefined, object, symbol.

2) Переменная в JavaScript может содержать любой тип данных (даже null и undefined).

3) JS это динамически типизированный язык (переменные не привязаны к типу данных).

number

Целые числа и числа с плавающей точкой.

Существуют специальные числовые значения (Infinity, -Infinity и NaN).

Математические операции в JS "безопасны", если поделим "A" на 0, то код не взорвётся, получим NaN.

BigInt

Тот же number, но только для бОльших чисел (произвольной длины).

string

Должна быть заключена в кавычки ('', "", `${userName}`).

Обратные кавычки (``), внутри {} принимают переменную или выражение и возвращают результат как часть строки.

bool

Может принимать только два значения: true (истина) и false (ложь).

null

Это просто отдельный тип, специальное значение, которое представляет собой «ничего», «пусто» или «значение неизвестно».

undefined

Это просто отдельный тип, специальное значение, означает, что «значение не было присвоено».

Если переменная объявлена, но ей не присвоено никакого значения, то её значением будет undefined.

object

Все остальные типы называются «примитивными», потому что их значениями могут быть только простые значения (будь то строка, или число, или что-то ещё). В объектах же хранят коллекции данных или более сложные структуры.

symbol

Используется для создания уникальных идентификаторов в объектах. P.S. вот тут не совсем понял, но говорят дальше подробнее будет.

Оператор typeof

Нужен если хотим сделать проверку на тип, возвращает строку с типом аргумента.

2.7 Преобразование типов

Строковое преобразование (к string)

1) Через alert(value).

2) Через String(value).

Численное преобразование (к number)

1) Через математические операторы (*, /, -).

2) Через Number(value).

3) Через унарный знак "+".

Логическое преобразование (к bool)

1) Через логические операции (a > b вернёт true/false, и так далее).

2) Через Boolean(value).

3) Через двойной восклицательный знак (!!something).

Дополнительно

0, null, undefined, NaN, "" возвращает false, остальное возвращает true (даже строка с пробелом " " вернёт true).

2.8 Базовые операторы, математика

Термины: «унарный», «бинарный», «операнд»

1) Операнд - то над чем совершается операция ( 5 * 2 )

2) Унарный применяется к одному операнду (например +value).

3) Бинарный применяется к двум операндам (например y - x).

Математика

1) Сложение (+)

2) Вычитание (-)

3) Умножение (*)

4) Деление (/)

5) Взятие остатка от деления (%, например 8%3 будет 2)

6) Возведение в степень (**, например 2**3 будет 8)

Сложение строк при помощи бинарного +

В JS можно сложить строки через знак + (например let s = "Hello" + "World!"; // Hello World!). Причём даже если операндов много и хоть один из них строка, то результатом будет строка.

Приоритет операторов

1) Большая часть тут понятна из школьной программы (* и / выше + и -, приоритет скобкам итд).

2) Приоритет унарных операторов выше, чем бинарных (+apples + +oranges, сначала приведёт к числу переменные, а потом произойдёт сложение).

3) У присваивания (=) низкий приоритет из-за таких (let x = 2*3;) моментов.

Присваивание

1) Т.к. присваивание это оператор, его можно использовать прямо в вычислении: let a = 1, b = 2; let c = 3 - (a = b + 1); // a === 3; b === 0.

2) Можно использовать присваивание по цепочке (let a, b, c; a = b = c = 4;), но лучше так не делать.

Сокращённая арифметика с присваиванием

Бывает нужно записать в переменную результат операции с ней же и ещё чем-то (let n = 5; n = n + 5;).

Для таких случаев используют сокращения (*=, +=, например let n = 5; n += 5;).

Инкремент/декремент

1) Инкремент увеличивает переменную на 1 (value++), декремент уменьшает на 1 (value--).

2) Можно использовать только на переменной, с числом кинет ошибку.

3) ++ и -- можно использовать как до, так и после переменной. Префиксная форма возвращает новое значение, в то время как постфиксная форма возвращает старое (до увеличения/уменьшения числа).

4) Использование ++ или -- меняет исходное значение переменной (let a = 0; let b = ++a; // a === 1; b === 1;) Если хочется тут же использовать результат, то нужна префиксная форма. Если нужно увеличить и при этом получить значение переменной до увеличения – нужна постфиксная форма

Оператор "запятая"

Позволяет вычислить несколько выражений, разделяя их запятой, но вернёт только результат последнего (например, let a = (1+2, 3+4); // a === 7).

2.9 Операторы сравнения

Результат сравнения имеет логический тип

1) Все операторы сравнения возвращают значение логического типа (true/false).
2) Результат сравнения можно присвоить переменной, как и любое значение (let a = 3 > 2; // true).

Сравнение строк

1) Строки сравниваются посимвольно, по алфавитному порядку ('Коты' > 'Кода' //true).
2) Имеет значение регистр символов. Заглавная буква "A" меньше строчной "a", т.к. строчные буквы имеют больший код в таблице символов.

Сравнение разных типов

При сравнении значений разных типов JavaScript приводит каждое из них к числу ("2" > 1 // true).

Строгое сравнение

1) Стоит использовать его (===), вместо обычного сравнения (==), т.к. он проверяет равенство без приведения типов ("1" === 1 // false).
2) Аналогично и с оператором строгого неравенства (!==).

Сравнение с null и undefined

Достаточно придерживаться двух правил:
1) Относитесь очень осторожно к любому сравнению с undefined/null, кроме случаев строгого равенства ===.
2) Не используйте сравнения >= > < <= с переменными, которые могут принимать значения null/undefined, разве что вы полностью уверены в том, что делаете.

2.11 Логические операторы

|| (ИЛИ)

Синтаксис: result = a || b; false только когда оба условия false.

ИЛИ «||» находит первое истинное значение

При выполнении ИЛИ || с несколькими значениями: result = value1 || value2 || value3;
Цепочка ИЛИ "||" возвращает первое истинное значение или последнее, если такое значение не найдено.
1) С помощью цепочки "||" можно найти первую переменную с данными (возвращает только первое подходящее значение).
2) Сокращённое вычисление: let x; (2 > 3) || (x = 1); // первый аргумент false, переключится на второй и задаст x значение 1.

&& (И)

Синтаксис: result = a && b; true только когда оба условия true.

И «&&» находит первое ложное значение

При нескольких подряд операторах И: result = value1 && value2 && value3;
Цепочка И "&&" возвращает первое ложное значение, или последнее, если ничего не найдено.
1) Приоритет оператора И && больше, чем ИЛИ ||, так что он выполняется раньше. a && b || c && d равноценен (a && b) || (c && d).
2) Действие в правой части && выполнится только в том случае, если до него дойдут вычисления.

! (НЕ)

Синтаксис: result = !value;
1) Оператор принимает один аргумент и: приводит его к bool, возвращает противоположное значение.
2) Двойное НЕ используют для преобразования к логическому типу: !!"something" // true

2.12 Оператор объединения с null '??'

Основы

Результат выражения a ?? b будет следующим: a, если значение a определено, b, если значение a не определено.
То есть оператор ?? возвращает первый аргумент, если он не null/undefined, иначе второй.
Пример: result = a ?? b равноценен result = (a !== null && a !== undefined) ? a : b;

Сравнение с ||

Важное различие между ними заключается в том, что:
1) || возвращает первое истинное (true) значение.
2) ?? возвращает первое определённое значение.

Оператор || не различает false, 0, пустую строку "" и null/undefined. Пример let height = 0;
1) height || 100 проверяет, имеет ли переменная height ложное значение. // 100
2) height ?? 100 проверяет, что переменная height содержит null/undefined. // 0

Приоритет

Оператор ?? имеет довольно низкий приоритет: 5. Таким образом, оператор ?? вычисляется до = и ?, но после большинства других операций, таких как +, *.

Из этого следует, что если нужно выбрать значение при помощи оператора ?? вместе с другими операторами в выражении, следует добавить круглые скобки.

Использование ?? вместе с && или ||

Запрещено использовать вместе с || или && без явно указанных круглых скобок.

2.13 Циклы while и for

Цикл «while»

Синтаксис: while (condition) {...}
Код из тела цикла выполняется, пока условие condition истинно.
Второй вариант использования: "пока условие true" (while (i) {i--;} // остановится на 0).

Цикл «do…while»

Синтаксис: do {...} while (condition) {...}
Цикл сначала выполнит тело, а затем проверит условие condition, и пока его значение равно true, он будет выполняться снова и снова.

Цикл «for»

Синтаксис: for (начало; условие; шаг) {...}
Начало выполняется один раз, а затем каждая итерация заключается в проверке условия, после которой выполняется тело и шаг.

1) Можно объявить переменую внутри цикла: for (let i = 10; i > 0; i--) {...}
2) Можно пропустить любую из частей for: for (;i>3;) {...} (аналогично while (i > 3)).

Прерывание цикла: «break»

Обычно цикл завершается при вычислении условия в false. Но можно выйти при помощи break;
while (true) {value = +prompt("value", ""); if (!value) {break;}}

Переход к следующей итерации: continue

Директива continue – «облегчённая версия» break. При её выполнении цикл не прерывается, а переходит к следующей итерации (если условие все ещё равно true).

Другими словами означает: пропустить оставшееся и пойти дальше.

Нельзя использовать continue вместе с тернарным оператором (будет ошибка).

Метки для break/continue

Например, есть for в котором ещё второй for. Нам нужно выйти из второго for на область видимости файла в котором написан первый for, для этого используют метки.

Синтаксис: labelName: for (...) {...} с последующим break labelName.
Перенесёт нас на уровень где был объявлен цикл с меткой, и перенесет нас в его конец.
Но метка не позволит прыгнуть куда угодно (метка должна находиться выше цикла, а break внутри цикла).

2.14 Конструкция "switch"

Синтаксис

Конструкция switch имеет один или более блок case и необязательный блок default.

switch(x) {case 'val1': ... break; case 'val2': ... break; default: ... break;}

switch выполняется от соответствующей директивы case и далее, до ближайшего break (или до конца switch). Если ни один case не совпал – выполняется (если есть) вариант default.

Если break нет, то выполнение пойдёт ниже по следующим case, при этом остальные проверки игнорируются.

Любое выражение может быть аргументом для switch/case

И switch и case допускают любое выражение в качестве аргумента.

switch (+a) {case b + 1: ... break;}

Группировка «case»

Несколько вариантов case, использующих один код, можно группировать.

switch(x) {case 'val1': case 'val2': ... break; default: ... break;}

Возможность группировать case – это побочный эффект того, как switch/case работает без break.

Тип имеет значение

Проверка на равенство всегда строгая. Значения должны быть одного типа, чтобы выполнялось равенство.

То есть если три case это строка, а четвертый это число, его код не выполнится.

Задачи. Напишите "if", аналогичный "switch"

if (browser === 'Edge') { alert( "You've got the Edge!" ); } else if (browser == 'Chrome' || browser == 'Firefox' || browser == 'Safari' || browser == 'Opera') { alert( 'Okay we support these browsers too' ); } else { alert( 'We hope that this page looks ok!' ); }

Задачи. Переписать условия "if" на "switch"

const number = +prompt('Введите число между 0 и 3', ''); switch(number) { case 0: alert('Вы ввели число 0'); break; case 1: { alert('Вы ввели число 1'); break; } case 2: case 3: { alert('Вы ввели число 2, а может и 3'); break; } }

2.15 Функции

Объявление функции

Синтаксис объявления: function showMessage() {...}. Синтаксис вызова: showMessage();

Ключевое слово function => имя функции => параметры в круглых скобках и код функции.

Локальные переменные

Переменные, объявленные внутри функции, видны только внутри этой функции (это фича блочной ОВ let/const).

Внешние переменные

У функции есть доступ к внешним переменным (объявленные выше тела функции, тоже фича области видимости).

1) Функция обладает полным доступом к внешним переменным и может изменять их значение.
2) Внешняя переменная используется, только если внутри функции нет такой локальной.

Глобальные переменные

Переменные, объявленные снаружи всех функций называются глобальными.

Глобальные переменные видимы для любой функции (если только их не перекрывают одноимённые локальные переменные). Желательно сводить использование глобальных переменных к минимуму.

Параметры

Мы можем передать внутрь функции любую информацию, используя параметры (также называемые аргументами функции).

Например function showMessage(from, text). Тут from и text - это параметры.

Параметры по умолчанию

Если параметр не указан, то его значением становится undefined.

Пример с параметром по умолчанию function showMessage(from, text "текст по умолчанию"). В качестве параметра по умолчанию можно использовать функцию, которая вернёт значение.

Возврат значения

Функция может вернуть результат, который будет передан в вызвавший её код.

Пример: function sum(a, b) { return a + b; } let result = sum(1, 2);

1) return может находиться в любом месте тела функции. Как только выполнение доходит до этого места, функция останавливается.
2) Вызовов return может быть несколько, например с условием if () {...}.
3) Возможно использовать return и без значения. Это приведёт к немедленному выходу из функции.
4) Результат функции с пустым return или без него – undefined.

Выбор имени функции

Функция – это действие. Поэтому имя функции обычно является глаголом. Оно должно быть простым, точным и описывать действие функции.

Как правило, используются глагольные префиксы, обозначающие общий характер действия, после которых следует уточнение. Примеры:

1) "get…" – возвращают значение, "calc…" – что-то вычисляют, "create…" – что-то создают, "check…" – что-то проверяют и возвращают логическое значение, и т.д.

Функция должна делать только то, что явно подразумевается её названием. И это должно быть одним действием. Два независимых действия обычно подразумевают две функции, даже если предполагается, что они будут вызываться вместе (в этом случае мы можем создать третью функцию, которая будет их вызывать). Пример: getAge – будет плохим выбором, если функция будет выводить alert с возрастом (должна только возвращать его).

Функции == Комментарии

Функции должны быть короткими и делать только что-то одно. Если это что-то большое, имеет смысл разбить функцию на несколько меньших.

Допустимо создавать функции, даже если мы не планируем повторно использовать их. Такие функции структурируют код и делают его более понятным.

Задачи. Обязателен ли "else"?

Отличий нет, если в конструкции if или множества if (вместо if else) есть return, то вместо else можно в конце добавить просто return.

2.16 Function Expression

Основы

Function Declaration: function sayHi() { alert( "Привет" ); }

Function Expression: let sayHi = function() { alert( "Привет" ); };

Функции-«колбэки»

Пример: есть функция function ask(question, yes, no) {if (confirm(question)) {yes()} else {no();}}. ask - текст, yes - функция, no - функция.

А после вызываем эту функцию со значениями которые являются некоторыми функциями: ask("Вы согласны?", showOk, showCancel);

Аргументы функции ask ещё называют функциями-колбэками или просто колбэками.

Ключевая идея в том, что мы передаём функцию и ожидаем, что она вызовется обратно (от англ. «call back» – обратный вызов) когда-нибудь позже, если это будет необходимо. В нашем случае, showOk становится колбэком’ для ответа «yes», а showCancel – для ответа «no» (см. пример выше).

Можно записать это короче, через анонимные функции: ask("Да?", function() {"Да"}, function() {"Нет"}).

Функция – это значение, представляющее «действие»

Обычные значения, такие как строки или числа представляют собой данные.

Функции, с другой стороны, можно воспринимать как «действия».

Мы можем передавать их из переменной в переменную и запускать, когда захотим.

Function Expression в сравнении с Function Declaration

Function Declaration: function sayHi() { alert( "Привет" ); }

Function Declaration можно использовать во всем скрипте (или блоке кода, если функция объявлена в блоке).

Имеет блочную ОВ. В строгом режиме, когда Function Declaration находится в блоке {...}, функция доступна везде внутри блока. Но не снаружи него.

Чтобы функция была видна вне блока, можно объявить переменную для функции вне блока, присвоить через Function Expression в блоке и вызвать вне блока.

Function Expression: let sayHi = function() { alert( "Привет" ); };

Function Expression создаётся, когда выполнение доходит до него, и затем уже может использоваться.

Когда использовать Function Declaration, а когда Function Expression?

В большинстве случаев, когда нам нужно создать функцию, предпочтительно использовать Function Declaration, т.к. функция будет видима до своего объявления в коде. Function Expression в случае, когда синтаксис Function Declaration не подходит для нашей задачи.

2.17 Функции-стрелки, основы

Основы

Синтаксис: let func = (arg1, arg2, ...argN) => expression

Функции-стрелки могут быть использованы так же, как и Function Expression, иметь коллбеки:
let welcome = (age < 18) ? () => alert('Привет') : () => alert("Здравствуйте!");

Многострочные стрелочные функции

Можно писать в несколько строк, но нужен будет return и скобки {}

2.18 Особенности JavaScript

Структура кода

Инструкции разделяются точкой с запятой.

Строгий режим

Чтобы по максимуму использовать возможности современного JavaScript, все скрипты рекомендуется начинать с добавления директивы "use strict".

Переменные

1) Можно объявить при помощи: let, const (константа, т.е. изменению не подлежит), var (устаревший способ).

2) Имя переменной может включать: Буквы и цифры, однако цифра не может быть первым символом. Символы $ и _ используются наряду с буквами. Иероглифы.

Операторы

1) Арифметические: Простые * + - /, а также деление по модулю % и возведение в степень **.

2) Операторы присваивания: Простые a = b и составные a *= 2.

3) Условный оператор: cond ? resultA : resultB.

4) Логические операторы: И &&, ИЛИ ||, НЕ !value.

5) Сравнение: Проверка на равенство == значений разных типов конвертирует их в число (за исключением null и undefined).

Циклы

1) Мы изучили три вида циклов: while(condition){...}, do {...} while (condition);, for(let i = 0; i < 10; i++) {...}

2) Переменная, объявленная в цикле for(let...), видна только внутри цикла.

3) Директивы break/continue позволяют выйти из цикла/текущей итерации. Используйте метки для выхода из вложенных циклов.

Конструкция «switch»

Конструкция «switch» может заменить несколько проверок if. При сравнении она использует оператор строгого равенства ===.

Функции

1) Function Declaration, Function Expression, Стрелочные функции.

2) У функций могут быть локальные переменные: т.е. объявленные в теле функции. Такие переменные видимы только внутри функции.

3) У параметров могут быть значения по умолчанию: function sum(a = 1, b = 2) {...}.

4) Функции всегда что-нибудь возвращают. Если нет оператора return, результатом будет undefined.
