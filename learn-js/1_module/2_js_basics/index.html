2.3 Строгий режим - "use strict"

Строка "use strict"; добавляет поддержку старого кода

Должна находиться в начале скрипта в котором может быть старый код. Обычно его используют на весь файл, но можно поставить внутри конкретной функции.

Например задание восьмеричных переменных.

(function getVariable() {
let result = 010; return console.log('2.3 getVariable 010', result);
})();
// Ответ (см. лог): 2.3 getVariable 010 8

(function getVariableUS() {
"use strict"; let result = 010; return console.log('2.3 getVariableUS 010', result);
})();
// Ответ (см. лог): Uncaught SyntaxError: Octal literals are not allowed in strict mode.

2.4 Переменные

let

Переменная объявляется через let, мы можем либо задать её сразу (let a = "a";), либо позже (let a; a = "a";).

Можем объявить несколько переменных в одной строке: (let a = "a", b = "b", age = 22;), но такой способ не рекомендуется.

let можно переопределять: (let a = "a"; a = "b";), старые данные удаляются.

Переменную let с именем можно задать всего один раз (let a = "A"; let a = "B"; => Ошибка).

var

Есть устаревший вид переменной let (var), которая имеет некоторые отлчия.

Переменную var с именем можно задать несколько раз (затерев старое значение) (var a = "A"; let a = "B"; => a === "B").

Пример блочной области видимости let

if (true) {
let hi = "Hi!"; console.log("2.4 let in", hi);
}
console.log("2.4 let out", hi);
// Ответ (см. лог): 2.4 let in Hi! и второй лог Uncaught ReferenceError: hi is not defined

Поведение счётчика цикла при var и let

Имена переменных

1) Имя переменной должно содержать только буквы, цифры или символы $ и _.

2) Первый символ не должен быть цифрой.

3) Обычно используют camelCase.

4) Регистр имеет значение (apple и AppLE – это две разные переменные)

5) Можно задавать переменные на любом языке (let 我 = '...';), но не рекомендуется P.S. лол об этом не знал :D

6) Есть список зарезервированные слов которыми нельзя назвать переменную (let, class, function итд)

7) Всегда стоит задавать переменные через let (нельзя делай так wordA = "A"; делай так let wordA = "A";)

const

Константы нельзя изменить. Имеют блочную область видимости, как и let.

P.S. давно слышал тему, что если константа это массив, мы можем менять отдельные элементы массива, но переопределять не можем. И ещё читал давно, чтобы полностью запретить изменение массива, используют метод freeze().

const

Константы нельзя изменить. Имеют блочную область видимости, как и let.

Константы в верхнем регистре

1) Используют как псевдонимы для «жёстко закодированных» значений: именования цвета итд (const COLOR_RED = "#F00").

2) Константы в camelCase используют во всех остальных случаях.

Придумывайте правильные имена

1) Название переменной должно иметь ясный и понятный смысл, говорить о том, какие данные в ней хранятся.

2) Используйте легко читаемые имена, такие как userName или shoppingCart.

3) Избегайте использования аббревиатур или коротких имён, таких как a, b, c.

4) Делайте имена максимально описательными и лаконичными. Примеры плохих имён: data и value. Такие имена ничего не говорят. Их можно использовать только в том случае, если из контекста кода очевидно, какие данные хранит переменная.

5) Договоритесь с вашей командой об используемых терминах. Если посетитель сайта называется «user», тогда мы должны называть связанные с ним переменные currentUser или newUser, а не, к примеру, currentVisitor или newManInTown.

2.5 Типы данных

Основы

1) Есть восемь основных типов данных в JavaScript: number, BigInt, string, bool, null, undefined, object, symbol.

2) Переменная в JavaScript может содержать любой тип данных (даже null и undefined).

3) JS это динамически типизированный язык (переменные не привязаны к типу данных).

number

Целые числа и числа с плавающей точкой.

Существуют специальные числовые значения (Infinity, -Infinity и NaN).

Математические операции в JS "безопасны", если поделим "A" на 0, то код не взорвётся, получим NaN.

BigInt

Тот же number, но только для бОльших чисел (произвольной длины).

string

Должна быть заключена в кавычки ('', "", `${userName}`).

Обратные кавычки (``), внутри {} принимают переменную или выражение и возвращают результат как часть строки.

bool

Может принимать только два значения: true (истина) и false (ложь).

null

Это просто отдельный тип, специальное значение, которое представляет собой «ничего», «пусто» или «значение неизвестно».

undefined

Это просто отдельный тип, специальное значение, означает, что «значение не было присвоено».

Если переменная объявлена, но ей не присвоено никакого значения, то её значением будет undefined.

object

Все остальные типы называются «примитивными», потому что их значениями могут быть только простые значения (будь то строка, или число, или что-то ещё). В объектах же хранят коллекции данных или более сложные структуры.

symbol

Используется для создания уникальных идентификаторов в объектах. P.S. вот тут не совсем понял, но говорят дальше подробнее будет.

Оператор typeof

Нужен если хотим сделать проверку на тип, возвращает строку с типом аргумента.

2.7 Преобразование типов

Строковое преобразование (к string)

1) Через alert(value).

2) Через String(value).

Численное преобразование (к number)

1) Через математические операторы (*, /, -).

2) Через Number(value).

3) Через унарный знак "+".

Логическое преобразование (к bool)

1) Через логические операции (a > b вернёт true/false, и так далее).

2) Через Boolean(value).

3) Через двойной восклицательный знак (!!something).

Дополнительно

0, null, undefined, NaN, "" возвращает false, остальное возвращает true (даже строка с пробелом " " вернёт true).

2.8 Базовые операторы, математика

Термины: «унарный», «бинарный», «операнд»

1) Операнд - то над чем совершается операция ( 5 * 2 )

2) Унарный применяется к одному операнду (например +value).

3) Бинарный применяется к двум операндам (например y - x).

Математика

1) Сложение (+)

2) Вычитание (-)

3) Умножение (*)

4) Деление (/)

5) Взятие остатка от деления (%, например 8%3 будет 2)

6) Возведение в степень (**, например 2**3 будет 8)

Сложение строк при помощи бинарного +

В JS можно сложить строки через знак + (например let s = "Hello" + "World!"; // Hello World!). Причём даже если операндов много и хоть один из них строка, то результатом будет строка.

Приоритет операторов

1) Большая часть тут понятна из школьной программы (* и / выше + и -, приоритет скобкам итд).

2) Приоритет унарных операторов выше, чем бинарных (+apples + +oranges, сначала приведёт к числу переменные, а потом произойдёт сложение).

3) У присваивания (=) низкий приоритет из-за таких (let x = 2*3;) моментов.

Присваивание

1) Т.к. присваивание это оператор, его можно использовать прямо в вычислении: let a = 1, b = 2; let c = 3 - (a = b + 1); // a === 3; b === 0.

2) Можно использовать присваивание по цепочке (let a, b, c; a = b = c = 4;), но лучше так не делать.

Сокращённая арифметика с присваиванием

Бывает нужно записать в переменную результат операции с ней же и ещё чем-то (let n = 5; n = n + 5;).

Для таких случаев используют сокращения (*=, +=, например let n = 5; n += 5;).

Инкремент/декремент

1) Инкремент увеличивает переменную на 1 (value++), декремент уменьшает на 1 (value--).

2) Можно использовать только на переменной, с числом кинет ошибку.

3) ++ и -- можно использовать как до, так и после переменной. Префиксная форма возвращает новое значение, в то время как постфиксная форма возвращает старое (до увеличения/уменьшения числа).

4) Использование ++ или -- меняет исходное значение переменной (let a = 0; let b = ++a; // a === 1; b === 1;) Если хочется тут же использовать результат, то нужна префиксная форма. Если нужно увеличить и при этом получить значение переменной до увеличения – нужна постфиксная форма

Оператор "запятая"

Позволяет вычислить несколько выражений, разделяя их запятой, но вернёт только результат последнего (например, let a = (1+2, 3+4); // a === 7).

2.9 Операторы сравнения

Результат сравнения имеет логический тип

1) Все операторы сравнения возвращают значение логического типа (true/false).
2) Результат сравнения можно присвоить переменной, как и любое значение (let a = 3 > 2; // true).

Сравнение строк

1) Строки сравниваются посимвольно, по алфавитному порядку ('Коты' > 'Кода' //true).
2) Имеет значение регистр символов. Заглавная буква "A" меньше строчной "a", т.к. строчные буквы имеют больший код в таблице символов.

Сравнение разных типов

При сравнении значений разных типов JavaScript приводит каждое из них к числу ("2" > 1 // true).

Строгое сравнение

1) Стоит использовать его (===), вместо обычного сравнения (==), т.к. он проверяет равенство без приведения типов ("1" === 1 // false).
2) Аналогично и с оператором строгого неравенства (!==).

Сравнение с null и undefined

Достаточно придерживаться двух правил:
1) Относитесь очень осторожно к любому сравнению с undefined/null, кроме случаев строгого равенства ===.
2) Не используйте сравнения >= > < <= с переменными, которые могут принимать значения null/undefined, разве что вы полностью уверены в том, что делаете.

2.11 Логические операторы

|| (ИЛИ)

Синтаксис: result = a || b; false только когда оба условия false.

ИЛИ «||» находит первое истинное значение

При выполнении ИЛИ || с несколькими значениями: result = value1 || value2 || value3;
Цепочка ИЛИ "||" возвращает первое истинное значение или последнее, если такое значение не найдено.
1) С помощью цепочки "||" можно найти первую переменную с данными (возвращает только первое подходящее значение).
2) Сокращённое вычисление: let x; (2 > 3) || (x = 1); // первый аргумент false, переключится на второй и задаст x значение 1.

&& (И)

Синтаксис: result = a && b; true только когда оба условия true.

И «&&» находит первое ложное значение

При нескольких подряд операторах И: result = value1 && value2 && value3;
Цепочка И "&&" возвращает первое ложное значение, или последнее, если ничего не найдено.
1) Приоритет оператора И && больше, чем ИЛИ ||, так что он выполняется раньше. a && b || c && d равноценен (a && b) || (c && d).
2) Действие в правой части && выполнится только в том случае, если до него дойдут вычисления.

! (НЕ)

Синтаксис: result = !value;
1) Оператор принимает один аргумент и: приводит его к bool, возвращает противоположное значение.
2) Двойное НЕ используют для преобразования к логическому типу: !!"something" // true

2.12 Оператор объединения с null '??'

Основы

Результат выражения a ?? b будет следующим: a, если значение a определено, b, если значение a не определено.
То есть оператор ?? возвращает первый аргумент, если он не null/undefined, иначе второй.
Пример: result = a ?? b равноценен result = (a !== null && a !== undefined) ? a : b;

Сравнение с ||

Важное различие между ними заключается в том, что:
1) || возвращает первое истинное (true) значение.
2) ?? возвращает первое определённое значение.

Оператор || не различает false, 0, пустую строку "" и null/undefined. Пример let height = 0;
1) height || 100 проверяет, имеет ли переменная height ложное значение. // 100
2) height ?? 100 проверяет, что переменная height содержит null/undefined. // 0

Приоритет

Оператор ?? имеет довольно низкий приоритет: 5. Таким образом, оператор ?? вычисляется до = и ?, но после большинства других операций, таких как +, *.

Из этого следует, что если нужно выбрать значение при помощи оператора ?? вместе с другими операторами в выражении, следует добавить круглые скобки.

Использование ?? вместе с && или ||

Запрещено использовать вместе с || или && без явно указанных круглых скобок.
