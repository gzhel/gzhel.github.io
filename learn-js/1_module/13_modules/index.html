13.1 Модули, введение

Основы

Модуль – это просто файл. Один скрипт – это один модуль.

export отмечает переменные и функции, которые должны быть доступны вне текущего модуля.

import позволяет импортировать функциональность из других модулей.

У модулей есть своя область видимости.

В модулях всегда включена директива use strict.

13.2 Экспорт и импорт

Основы

Директивы экспорт и импорт имеют несколько вариантов вызова.

Для export:

1) Перед объявлением класса/функции/…: export [default] class/function/variable ...

2) Отдельный от объявления экспорт: export {x [as y], ...}.

3) Реэкспорт (импортировать что-то и тут же экспортировать): export {sayHi} from './say.js';


Для import:

1) Именованные экспорты из модуля: import {x [as y], ...} from "module"

2) Импорт по умолчанию: import x from "module" (для export default)

3) Всё сразу: import * as obj from "module"

4) Только подключить модуль (его код запустится), но не присваивать его переменной: import "module"
